# MW-15 최종 모바일 QA 기록

이 문서는 배포 전후 최종 모바일 QA를 기록하기 위한 체크 문서입니다.

## 자동 점검 완료 항목

- `pnpm lint`로 정적 검사를 수행합니다.
- `pnpm build`로 production build와 타입 검사를 수행합니다.
- `public/images/hero.jpg`, `public/images/og-image.jpg`, `gallery-*.jpg` 기본 파일을 준비해 이미지 경로 404를 줄입니다.
- Spring Boot, DB, API 서버, Prisma/ORM, 관리자 화면은 추가하지 않습니다.

## 실기기 확인 환경

아래 항목은 배포 URL에서 실제 기기로 확인합니다.

| 환경 | 확인 항목 | 결과 |
| --- | --- | --- |
| iPhone Safari | 360px~430px 레이아웃, 인트로 전환, 스크롤 | 미확인 |
| Android Chrome | 갤러리 모달, 계좌 복사, 캘린더 저장, 링크 복사 | 미확인 |
| 카카오톡 인앱 브라우저 | 링크 열림, 공유 fallback, OG 미리보기 | 미확인 |
| 네이버앱/삼성인터넷 | 지도 이동, 공유 버튼, 복사 기능 | 미확인 |

## 기능별 체크

- [ ] 360px, 390px, 430px 화면에서 가로 스크롤이 없다.
- [ ] 카카오톡에서 배포 링크가 열린다.
- [ ] 카카오톡/문자 공유 미리보기에 제목, 설명, 이미지가 표시된다.
- [ ] 카카오맵, 네이버지도, 티맵, 구글맵 버튼이 정상 이동한다.
- [ ] 계좌번호 복사 후 완료 피드백이 표시된다.
- [ ] 캘린더 저장 시 `.ics` 파일이 생성된다.
- [ ] 갤러리 사진을 열고 닫을 수 있다.
- [ ] 실제 전화번호/계좌번호/주소 공개 여부를 최종 확인했다.

## 배포 전 주의사항

- `src/data/wedding.ts`의 `meta.url`은 실제 배포 URL로 교체해야 합니다.
- `public/images`의 기본 placeholder 이미지는 실제 사진으로 교체하는 것을 권장합니다.
- 카카오 공유를 활성화하려면 배포 환경변수 `NEXT_PUBLIC_KAKAO_JAVASCRIPT_KEY`를 설정합니다.
